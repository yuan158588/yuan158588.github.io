Git多个账号使用SSH Key

步骤

  • ```ruby
    $ ssh-keygen -t rsa -C “youremail1@xxx.com“

  • 分别配置公钥rsa_pub 到 github/gitee 的 setting SSH Key 中

  • 测试配置成功

Github Pages

使用规则

  • 参考Github pages + hexo 博客搭建教程

  • 首先得知道Github pages的规则：每个Github账号（比如Username）下面只能建立一个Pages，且命名必须符合这样的规则：”username/username.github.io”。创建成功后，username.github.io就是你的域名（当然你可以通过别名解析绑定自己的域名）

步骤

  • 创建仓库 username.github.io

  • 此时通过 https://username.github.io/ 就可以访问 Github Pages 了

    注: 这里可能不会立马刷新出来, 需要等待一会(具体时间不清楚)

  • 注意: 2020年10月1日，新创建的存储库将默认为“main”。

    因为后面 hexo 通过 hexo d 提交到 github 上的 master 分支,(应该是可以配置,暂时还没接触到,后面发现再说)

    会造成访问 github pages 以为 hexo 上传失败了,其实只需要在 setting -> github pages -> source 中切换即可

  • 能够正常访问, 此时 github pages 就部署并测试通过.

Hexo

​ 其实只是用 Github Pages 已经可以发布博客了,但是比较麻烦, 因此 Hexo 工具就应运而生了.

安装 hexo 步骤

  • Git + Node.js 依赖, 使用下面命令查看安装是否成功

  • 安装 hexo 命令

  • 验证安装是否成功 hexo -v

  • 至此, hexo 环境已经安装完成了.

生成 blog

  • 注意: 以下命令最好在 git bash 中执行,, 不然可能有这样那样的问题

  • 找一个存放 blog 的盘, 比如 D:

  • 在该路径下打开 cmd: hexo init blog

    此时会在该路径下生成 blog 文件夹

  • 修改 _config.yml 配置文件

  • 此时配置修改完成

  • 上传 hexo 需要插件deployer的支持

  • 先使用 hexo s 在本地预览, http://localhost:4000 查看 hexo 部署是否成功.

  • 此时使用 hexo d 就可以上传到 github/gitee 仓库了

    显示INFO Deploy done: git就说明上传成功了！

  • 此时在访问 https://username.github.io/ 就可以看到和 http://localhost:4000 一样的效果了.

  • 注意: 使用 https 执行 hexo d 每次都要登录, 将_config.yml里的repo 改成SSH（git@github.com）的链接就行了

总结

  • 主要分为两块内容 github pages 以及 hexo.

  • 利用 github pages 来实现静态blog, 使用 hexo 来管理blog.

  • 每步完成之后测试是否成功, 出现问题在慢慢定位在哪

  • 中间涉及到 github 的使用,以及 SSH Key 的配置, 尤其是有多个 github 账号时需要切换.

  • hexo 的使用后续在学习,先搭好框架

  • 后续在选择一个合适的主题
